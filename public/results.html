Key Themes
What worked well. Where it fell short.

What Worked Well

  • +NotebookLM for research — uploading RFIs and querying them with natural language was transformative for production design teams
  • +Image generation (Nano Banana Pro) produced high-quality, realistic reference images for character design, set concepts, and mood boards
  • +Chat in "thinking mode" delivered highly accurate code and debugging assistance with low hallucination
  • +Script analysis and document breakdowns completed in seconds versus days
  • +Multi-language translation of SRT files was stable and reliable, outperforming Copilot for large files
  • +Character design workflow cut from 5 days to 2 days (60% time savings)

What Fell Short

  • −Video generation (Veo) was unreliable — low resolution, context loss between prompts, 8-second limit, poor character consistency
  • −Image iteration frustrating — editing one element regenerated the entire image rather than adjusting in place
  • −Deep Research hallucinated specific names, page numbers, and citations that did not exist
  • −No integration with existing tools — no SharePoint connectors, no API access, web-only with no mobile app
  • −Access issues — sessions expired overnight, required specific links, some users couldn't determine if they had access
  • −Results sometimes inferior to Copilot, ChatGPT, or Claude for the same prompts

Recommendations
What the data suggests

Keep and Expand

  • 1NotebookLM — The standout feature. Teams want to connect it to SharePoint and RFI libraries for organization-wide knowledge access.
  • 2Image Generation — Already integrated into daily workflows for character design, set concepts, and mood boards. Higher resolution output is the top request.
  • 3Chat / Thinking Mode — Valued for coding, script analysis, and document breakdowns. Users want API access and connector support.

Address Before Scaling

  • 1Video generation — Not production-ready. Users recommend evaluating alternatives (Kling, Seedance 2.0) alongside Veo.
  • 2Onboarding and access — Simplify login flow, provide role-specific use cases, and create video tutorials.
  • 3Integration gaps — No SharePoint, no API, no mobile app. These limit adoption for daily use.
  • 4Hallucination risk — Provide training on verification practices, especially for research-heavy roles.
